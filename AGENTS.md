# Правила для агентов

## Ветки и интеграция

- Вся разработка ведётся в ветке `develop`.
- После завершения задачи агент обязан добавить или обновить unit-тесты для изменённой логики и убедиться, что они проходят.
- Только после успешной проверки задача может быть слита из `develop` в `main`.

## Коммиты

- Сообщение коммита кратко и точно описывает выполненное изменение.
- В сообщениях коммитов нельзя упоминать имена, идентификаторы или роли агентов.
